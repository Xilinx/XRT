.. _xrt_api_buffer:

Buffer objects (``xrt::bo``)
============================

**Header:** ``xrt/xrt_bo.h``

**Purpose.** Allocate and manage buffer objects (BOs) for host–device data movement and kernel arguments.

**C typedefs**

* ``xrtBufferHandle`` — opaque buffer handle
* ``xrtBufferFlags`` — BO flags (see ``xrt/detail/xrt_mem.h`` / XCL flag compatibility)
* ``xrtMemoryGroup`` — memory bank / group id
* ``struct xcl_buffer_handle`` — typed wrapper for ``xclBufferHandle`` when constructing ``xrt::bo`` from a raw buffer handle

**C++ supporting types**

* ``xrt::memory_group`` — alias for ``xrtMemoryGroup``
* ``xrt::pid_type`` — wrapper for ``pid_t`` to disambiguate constructors that take a process id
* ``xrt::bo::flags`` — ``normal``, ``cacheable``, ``device_only``, ``host_only``, ``p2p``, ``svm``, ``carveout``, …
* ``xrt::bo::export_handle`` — platform-specific export type (fd on Linux)
* ``xrt::bo::async_handle`` — pimpl handle; ``wait()`` for async BO operations

**``xrt::bo`` — main operations (categories)**

* **Allocation:** constructors from ``xrt::device`` (and optional ``xrt::hw_context`` where supported), size, memory group, flags; from user pointer; sub-buffers; import from export handle; alignment helpers as documented in header.
* **Data transfer:** ``read`` / ``write`` (with size/offset overloads), ``sync`` (direction flags from ``xrt_mem.h`` / XCL), ``map<T*>()`` for host mapping.
* **Peer / export:** ``export_buffer()`` and import constructors.
* **Info:** ``size()``, address helpers, and related accessors in header.
* **Async:** asynchronous sync/copy variants returning ``async_handle`` where available.

**Free functions / C API**

The header exposes a large set of ``xrtBONamedAllocate*``, ``xrtBOSubBuffer``, ``xrtBOExport``, ``xrtBOMap``, ``xrtBOSync``, etc., mirroring the C++ functionality for C host code—see the bottom of ``xrt_bo.h`` for exact symbol names and parameters.

**Include**

.. code-block:: cpp

   #include <xrt/xrt_bo.h>
