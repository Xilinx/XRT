Device and UUID
===============

``xrt/xrt_device.h`` — ``xrt::device``
--------------------------------------

**Purpose.** Open an accelerator, query properties, load or register xclbin metadata, and access the underlying shim where needed.

**C++ type**

* ``xrt::device`` — device handle; default construction is only valid for assignment; use index, BDF string, or ``xclDeviceHandle`` constructor to open a device.

**``xrt::info::device`` (with ``xrt::device::get_info<Param>()``)** — device information parameters. The return type is fixed by param traits at compile time. Enumerators include (among others) ``platform``, ``host``, ``name``, ``bdf``, ``interface_uuid``, ``kdma``, ``m2m``, ``nodma``, ``offline``, sensor strings, and AIE-related info fields; several are marked deprecated in the header—prefer the strings documented in the current release notes for your platform.

**Key members (C++)**

* Constructors: from unsigned index, from ``std::string`` (BDF or index), from ``xclDeviceHandle``.
* ``template <info::device param> … get_info() const`` — typed device information.
* ``xrt::uuid register_xclbin(const xrt::xclbin&)`` — register an xclbin without binding hardware (used with hardware contexts).
* ``xrt::uuid load_xclbin(…)`` — **deprecated** in favor of ``xrt::hw_context``; overloads: ``axlf*``, file path, ``xrt::xclbin`` object.
* ``xrt::uuid get_xclbin_uuid() const`` — UUID of the xclbin on the device.
* ``get_xclbin_section<SectionType>(axlf_section_kind, const xrt::uuid&)`` — retrieve a section from the loaded xclbin.
* ``operator==`` / ``operator!=`` — same physical device comparison.

**C API (``extern "C"`` in the same header)**

* ``xrtDeviceOpen(unsigned int index)`` → ``xrtDeviceHandle``
* ``xrtDeviceOpenByBDF(const char* bdf)``
* ``xrtDeviceOpenFromXcl(xclDeviceHandle)``
* ``xrtDeviceClose(xrtDeviceHandle)``
* ``xrtDeviceLoadXclbin``, ``xrtDeviceLoadXclbinFile``, ``xrtDeviceLoadXclbinHandle``, ``xrtDeviceLoadXclbinUUID``
* ``xrtDeviceGetXclbinUUID``
* ``xrtDeviceToXclDevice`` — access to legacy shim handle

**Typedef**

* ``xrtDeviceHandle`` — opaque ``void*``


``xrt/xrt_uuid.h`` — ``xrt::uuid``
----------------------------------

**Purpose.** Value type wrapping the platform UUID type (``xuid_t`` on Linux) for xclbin and shell identity.

**C++ type**

* ``xrt::uuid`` — default (cleared), from ``xuid_t``, from UUID string, copy/move, ``get()`` for raw id, comparison operators, ``to_string()``, etc. (see header for full list).

**Include**

.. code-block:: cpp

   #include <xrt/xrt_device.h>
   #include <xrt/xrt_uuid.h>
